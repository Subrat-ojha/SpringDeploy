1. What is the difference between JDK and JRE, JVM and JIT Compiler?
JDK (Java Development Kit) is the complete development environment including JRE plus development tools. JRE (Java Runtime Environment) includes JVM and libraries needed to run Java applications. JVM (Java Virtual Machine) executes bytecode. JIT (Just-In-Time) compiler is part of JVM that improves performance by compiling bytecode to native machine code at runtime.
2. What is Java Virtual Machine (JVM)?
JVM is an abstract computing machine that provides runtime environment to execute Java bytecode. It converts Java bytecode into machine language and executes it. JVM is platform dependent and performs memory management, garbage collection, and security.
3. What are the different types of memory areas allocated by JVM?
JVM allocates memory in: 1) Method Area (class structures), 2) Heap (objects), 3) Java Stacks (thread execution), 4) PC Registers (thread instruction pointers), and 5) Native Method Stacks (native method calls).
4. What is JIT compiler?
JIT (Just-In-Time) compiler is part of JVM that improves Java performance by compiling bytecode to native machine code at runtime. It compiles frequently executed code paths (hot spots) to native code for faster execution.
5. How Java platform is different from other platforms?
Java platform is software-only platform that runs on top of hardware platforms. Unlike other platforms that are either hardware or OS, Java platform consists of JVM and Java API, providing "write once, run anywhere" capability.
6. What is platform independency in java?
Platform independency means Java programs can run on any platform without recompilation. This is achieved by compiling Java code to bytecode (platform-neutral) which is then interpreted by JVM (platform-specific).
7. How many class loaders in java?
Java has three built-in class loaders: 1) Bootstrap ClassLoader (loads core Java classes), 2) Extension ClassLoader (loads extension classes), and 3) Application/System ClassLoader (loads application classes from classpath).
8. What is delegation Hierarchy Algorithm?
The delegation hierarchy algorithm is the process where a class loader delegates class loading request to its parent before attempting to load the class itself. This ensures core Java classes cannot be overridden by application classes.
9. Can we write main method as public void static instead of public static void?
No, the main method must be declared as "public static void main(String[] args)". The order of "static" and "void" cannot be changed as it's part of Java language specification for the entry point of a program.
10. In Java, if we do not specify any value for local variables, then what will be the default value of the local variables?
Local variables in Java don't have default values. They must be initialized before use. Unlike instance variables (which get default values like 0, null, false), using an uninitialized local variable causes compile-time error.
11. Let say, we run a java class without passing any arguments. What will be the value of String array of arguments in Main method?
The String array will not be null, but it will be empty (length 0). Java always creates the args array even when no arguments are passed.
12. What is the difference between byte and char data types in Java?
byte is a signed 8-bit integer (-128 to 127), while char is an unsigned 16-bit Unicode character (0 to 65535). byte is used for raw binary data, char for text characters.
13. Ascending order of numeric data types?
byte (8-bit) < short (16-bit) < int (32-bit) < long (64-bit) < float (32-bit floating) < double (64-bit floating)
14. Can I take multi classes in single java file?
Yes, but only one class can be public and must match the filename. Other classes can be package-private (no access modifier) and used within the same file.
15. OOPS
OOPS stands for Object-Oriented Programming System. It's a programming paradigm based on objects containing data (attributes) and behavior (methods), with principles like encapsulation, inheritance, polymorphism, and abstraction.
16. What are the main principles of Object Oriented Programming?
The four main principles are: 1) Encapsulation (data hiding), 2) Inheritance (code reuse), 3) Polymorphism (many forms), and 4) Abstraction (hiding complexity).
17. What is the difference between Object Oriented Programming language and Object Based Programming language?
Object-Oriented languages support all OOP principles (inheritance, polymorphism, etc.), while Object-Based languages support objects but lack inheritance and polymorphism (e.g., JavaScript before ES6).
18. In Java what is the default value of an object reference defined as an instance variable in an Object?
The default value is null for all object references (non-primitive types). For primitive instance variables: numeric types default to 0, boolean to false, char to '\u0000'.
19. Why do we need constructor in Java?
Constructors are needed to: 1) Initialize object state, 2) Allocate resources, 3) Set default values, 4) Perform validation, and 5) Provide different ways to create objects (overloading).
20. Why do we need default constructor in Java classes?
Default constructor (no-arg) is needed when: 1) No other constructors are defined, 2) Framework requires it (e.g., JPA, Spring), 3) For subclass instantiation, 4) Reflection API uses it to create instances.
21. What is the value returned by Constructor in Java?
Constructors don't return any value (not even void). They implicitly return the newly created instance of the class. The 'new' keyword handles the assignment of the returned object reference.
22. Can we inherit a Constructor?
No, constructors are not inherited in Java. Each subclass must define its own constructors, though it can call superclass constructors using super().
23. Why constructors cannot be final, static, or abstract in Java?
Constructors cannot be: final (they initialize objects, no overriding needed), static (they create instance-specific objects), or abstract (they must have implementation to create objects).
24. What is the purpose of 'this' keyword in java?
'this' refers to the current object instance. It's used to: 1) Differentiate instance variables from parameters, 2) Call other constructors (this()), 3) Pass current object as parameter, 4) Return current object.
25. Explain the concept of a static variable. How is it different from an instance variable?
Static variable: Class-level, shared by all instances, loaded when class loads. Instance variable: Per-object state, each instance has its own copy. Static variables use less memory but can cause thread-safety issues.
26. Explain the concept of Inheritance?
Inheritance allows a class (subclass) to acquire properties and behaviors of another class (superclass). It enables code reuse and polymorphism. In Java, it's achieved using 'extends' keyword (single inheritance for classes).
27. Which class in Java is superclass of every other class?
Object class (java.lang.Object) is the superclass of all classes in Java, either directly or indirectly. It provides basic methods like equals(), hashCode(), toString(), etc.
28. Why Java does not support multiple inheritance?
Java avoids multiple inheritance of classes to prevent the "diamond problem" (ambiguity when same method exists in multiple parent classes). Instead, it supports multiple interface inheritance.
29. In OOPS, what is meant by composition?
Composition is a "has-a" relationship where a class contains references to other objects. It provides stronger encapsulation than inheritance and allows runtime behavior changes by changing component objects.
30. How aggregation and composition are different concepts?
Both are "has-a" relationships. Composition: Strong relationship, child can't exist without parent (e.g., Room in House). Aggregation: Weak relationship, child can exist independently (e.g., Student in School).
31. Why there are no pointers in Java?
Java doesn't have explicit pointers to: 1) Ensure security by preventing memory access violations, 2) Simplify memory management with garbage collection, 3) Make language more robust and less prone to errors.
32. What is encapsulation in Java, and why is it considered a fundamental principle of object-oriented programming?
Encapsulation is bundling data (variables) and code (methods) together while hiding implementation details. It's fundamental because: 1) Protects object integrity, 2) Reduces complexity, 3) Allows flexible implementation changes, 4) Improves maintainability.
33. What is the purpose of 'super' keyword in java?
'super' is used to: 1) Call superclass constructor (super()), 2) Access superclass methods/fields when overridden in subclass, 3) Resolve naming conflicts between superclass and subclass members.
34. Is it possible to use this() and super() both in same constructor?
No, they must be the first statement in a constructor, so only one can be used. this() calls another constructor in same class, super() calls parent class constructor.
35. What is the meaning of object cloning in Java?
Object cloning creates an exact copy of an object. It's achieved by implementing Cloneable interface and overriding clone() method. Shallow copy (default) copies field values, deep copy creates new objects for reference fields.
36. In Java, why do we use static variable?
Static variables are used for: 1) Class-level constants (final static), 2) Sharing data across all instances, 3) Maintaining count of instances, 4) Utility methods that don't need instance state.
37. Why it is not a good practice to create static variables in Java?
Overusing static variables can: 1) Cause thread-safety issues, 2) Make code harder to test (global state), 3) Reduce flexibility (tight coupling), 4) Increase memory usage (live for application lifetime).
38. What is the purpose of static method in Java?
Static methods are used for: 1) Utility/helper functions (Math.sqrt()), 2) Factory methods, 3) Operations that don't need instance state, 4) Main method entry point. They can be called without creating an instance.
39. Why do we mark main method as static in Java?
main() is static because: 1) It needs to be called before any objects exist, 2) JVM can call it without creating an instance, 3) It serves as the entry point that's independent of object state.
40. In what scenario do we use a static block?
Static blocks are used for: 1) Initializing static variables that need complex logic, 2) Loading native libraries, 3) One-time class initialization tasks, 4) Setting up static resources before any instance is created.
41. Is it possible to execute a program without defining a main() method?
Prior to Java 7, yes using static blocks (but program exits after). From Java 7 onwards, main() is strictly required for application execution. Applets/servlets have different entry points.
42. What happens when static modifier is not mentioned in the signature of main method?
The program will compile but won't run. JVM looks specifically for "public static void main(String[])" as the entry point. Non-static main would require an instance, but JVM can't create one before calling main.
43. What is the difference between static method and instance method in Java?
Static methods: Belong to class, called via ClassName.method(), can only access static members. Instance methods: Belong to objects, called via instance.method(), can access both instance and static members.
44. What is the other name of Method Overloading?
Method overloading is also called compile-time polymorphism or static polymorphism because the method to call is determined at compile time based on arguments.
45. How will you implement method overloading in Java?
Method overloading is implemented by: 1) Different number of parameters, 2) Different parameter types, 3) Different parameter order. Methods must have same name but different parameter lists.
46. What kinds of argument variations are allowed in Method Overloading?
Allowed variations: 1) Number of parameters, 2) Parameter types, 3) Parameter order. Not allowed: 1) Just return type change, 2) Just parameter names change, 3) Just exception list change.
47. Why it is not possible to do method overloading by changing return type of method in java?
Because method calls can ignore return values ("method();"), making calls ambiguous. The compiler wouldn't know which method to call when return type is the only difference.
48. Is it allowed to overload main() method in Java?
Yes, main() can be overloaded like any other method. However, only "public static void main(String[])" is called by JVM. Other overloaded versions must be called explicitly.
49. How do we implement method overriding in Java?
Method overriding requires: 1) Inheritance relationship, 2) Same method signature (name, parameters, return type), 3) Subclass method cannot be more restrictive (access modifier), 4) @Override annotation recommended.
50. Are we allowed to override a static method in Java?
No, static methods cannot be overridden - they can only be hidden (redefined in subclass). This is because static methods are bound at compile time (static binding) while instance methods use runtime binding.
51. Why Java does not allow overriding a static method?
Static methods belong to the class, not instances, so they're resolved at compile time (static binding). Overriding depends on runtime polymorphism which requires instance methods (dynamic binding).
52. Is it allowed to override an overloaded method?
Yes, an overloaded method (same name, different parameters) can be overridden in a subclass if it meets overriding rules. Overloading and overriding are independent concepts.
53. What is the difference between method overloading and method overriding in Java?
Overloading: Same class, same name, different parameters, compile-time binding. Overriding: Subclass, same signature, runtime binding. Overloading adds methods, overriding changes behavior.
54. Does Java allow virtual functions?
All non-static, non-private methods in Java are virtual by default (can be overridden). To prevent overriding: 1) Make method final, 2) Make class final, 3) Use private/static modifier.
55. What is meant by covariant return type in Java?
Covariant return allows overriding method to return a subtype of the original return type. Introduced in Java 5, it enables more specific return types in overridden methods.
56. What is Runtime Polymorphism?
Runtime polymorphism (dynamic method dispatch) is when JVM determines which method to call at runtime based on actual object type (not reference type). Achieved through method overriding.
57. Is it possible to achieve Runtime Polymorphism by data members in Java?
No, polymorphism applies only to methods, not fields. Field access is always resolved at compile time based on reference type (static binding), regardless of actual object type.
58. Explain the difference between static and dynamic binding?
Static binding (compile-time): Resolved using reference type (fields, static methods, private/final methods). Dynamic binding (runtime): Resolved using actual object type (overridden instance methods).
59. What is Abstraction in Object Oriented programming?
Abstraction is hiding complex implementation details and showing only essential features. In Java, achieved via abstract classes (partial abstraction) and interfaces (full abstraction).
60. How is Abstraction different from Encapsulation?
Abstraction focuses on hiding complexity and showing essentials (what). Encapsulation focuses on bundling data and methods while restricting access (how). Abstraction is achieved through encapsulation.
61. What is an abstract class in Java?
An abstract class: 1) Can't be instantiated, 2) May contain abstract methods (no body), 3) Can have concrete methods, 4) Used as base class, 5) Subclasses must implement abstract methods unless they're also abstract.
62. Is it allowed to mark a method abstract method without marking the class abstract?
No, if a class has any abstract methods, the class itself must be declared abstract. Only abstract classes can contain abstract methods (except for interface methods which are implicitly abstract).
63. Is it allowed to mark a method abstract as well as final?
No, these modifiers conflict. abstract requires the method to be overridden, final prevents overriding. The compiler will give an error if both are used together.
64. Can we instantiate an abstract class in Java?
No, we cannot directly instantiate an abstract class. But we can: 1) Create anonymous subclass instances, 2) Instantiate concrete subclasses, 3) Use factory methods that return subclass instances.
65. What is an interface in Java?
An interface is a reference type that contains: 1) Abstract methods (implicitly public abstract), 2) Default methods (Java 8+), 3) Static methods (Java 8+), 4) Constant variables (public static final). It defines a contract.
66. Is it allowed to mark an interface method as static?
Yes, since Java 8 interfaces can have static methods. These methods belong to the interface (not implementing classes) and must have a body. They're called via InterfaceName.method().
67. Why an Interface cannot be marked as final in Java?
final means cannot be extended, but the whole purpose of interfaces is to be implemented by classes. Making an interface final would make it useless as it couldn't be implemented.
68. What is a marker interface?
A marker interface is an empty interface used to mark classes with special meaning (e.g., Serializable, Cloneable). It provides runtime type information but no methods. Largely replaced by annotations now.
69. What can we use instead of Marker interface?
Annotations (introduced in Java 5) are the preferred alternative to marker interfaces. They provide more flexibility and can include metadata. Examples: @Override, @Deprecated, custom annotations.
70. How Annotations are better than Marker Interfaces?
Annotations are better because: 1) Can have parameters, 2) More flexible application (@Target), 3) Better tooling support, 4) Don't affect class hierarchy, 5) Can be processed at compile time.
71. What is the difference between abstract class and interface in Java?
Abstract class: Can have state (fields), constructors, any access modifiers. Interface: No state (only constants), no constructors, all methods public. Since Java 8: interfaces can have default/static methods.
72. Does Java allow us to use private and protected modifiers for variables in interfaces?
No, all interface variables are implicitly public static final (constants). All methods are public. Since Java 9: private methods are allowed (for helper methods in default methods).
73. How can we cast to an object reference to an interface reference?
If the object's class implements the interface, you can cast directly: Interface i = (Interface)object;. If unsure, use instanceof check first. ClassCastException occurs if object doesn't implement interface.
74. How can you change the value of a final variable in Java?
Normally final variables cannot be changed. Exceptions: 1) Blank final variables can be initialized once in constructor, 2) Final object references can have their internal state changed (not the reference itself).
75. Can a class be marked final in Java?
Yes, a final class cannot be extended (no subclasses). Examples from Java API: String, Integer, Double etc. Reasons to make class final: 1) Security, 2) Immutability, 3) Prevent behavior changes.
76. How can we create a final method in Java?
Add final modifier to method: "public final void method()". Final methods cannot be overridden by subclasses. Used to: 1) Prevent behavior changes, 2) Allow compiler optimizations, 3) Security critical methods.
77. How can we prohibit inheritance in Java?
Three ways: 1) Declare class as final, 2) Make all constructors private, 3) Use sealed classes (Java 17+). Final is most common approach.
78. Why Integer class in final in Java?
Integer is final to: 1) Ensure immutability (critical for caching, thread-safety), 2) Maintain consistent behavior (e.g., == comparison), 3) Security (prevent malicious subclassing), 4) Allow JVM optimizations.
79. What is a blank final variable in Java?
A final variable declared without initialization. Must be initialized exactly once before use: 1) In constructor (instance variables), 2) In static block (static variables), 3) At declaration.
80. How can we initialize a blank final variable?
Instance blank finals: Initialize in constructor. Static blank finals: Initialize in static block. Local blank finals: Must be initialized before use. All must be initialized exactly once.
81. Is it allowed to declare main method as final?
Yes, you can declare main method as final (e.g., "public static final void main(String[])"). It doesn't affect program execution since main() isn't overridden anyway. However, it's unconventional.
82. What is the purpose of package in Java?
Packages are used to: 1) Organize related classes, 2) Prevent naming conflicts, 3) Control access (package-private), 4) Provide namespace management, 5) Enable package-level documentation and annotations.
83. What is java.lang package?
java.lang is the most fundamental package, automatically imported. Contains core classes like: Object, String, System, Math, wrapper classes (Integer), exceptions, and threading classes.
84. Which is the most important class in Java?
Object class (java.lang.Object) is most important as it's the root of class hierarchy. Provides basic methods like: equals(), hashCode(), toString(), clone(), finalize(), getClass() etc.
85. Is it mandatory to import java.lang package every time?
No, java.lang is automatically imported by the compiler for all Java programs. Its classes (String, System, Object etc.) can be used directly without explicit import.
86. Can you import same package or class twice in your class?
Yes, but redundant imports are ignored by compiler. Multiple imports of same class/package don't cause errors but are unnecessary. Modern IDEs typically optimize imports to remove duplicates.
87. What is a static import in Java?
Static import (since Java 5) allows using static members (methods/fields) without class qualification. Syntax: "import static package.Class.*;" or "import static package.Class.member;". Use sparingly to avoid confusion.
88. What is the difference between import static com.test.Foo class and import com.test.Foo class?
import com.test.Foo: Imports class Foo for normal use (Foo.method()). import static com.test.Foo: Imports static members of Foo to use directly (method() instead of Foo.method()).
89. What is Locale in Java?
Locale (java.util.Locale) represents a specific geographical, political, or cultural region. Used for internationalization (i18n) to format dates, numbers, currencies etc. according to regional conventions.
90. How will you use a specific Locale in Java?
Create Locale object: 1) new Locale("en", "US"), 2) Locale.US (predefined). Use with formatters: NumberFormat.getInstance(Locale.FRANCE), DateFormat
91. What is serialization?
Serialization is the process of converting an object's state (including its data and relationships) into a byte stream that can be persisted to storage or transmitted across a network.
92. What is the purpose of serialization?
Serialization enables: 1) Object persistence, 2) Network transmission of objects, 3) Deep copying of objects, 4) Communication between JVMs, 5) Storing session data in web applications.
93. What is Deserialization?
Deserialization is the reverse process of serialization - reconstructing an object from its byte stream representation, restoring it to its original state in memory.
94. What is Serialization and Deserialization conceptually?
Conceptually, serialization is like packing an object for storage/transit, while deserialization is unpacking it. It allows objects to outlive the JVM process that created them.
95. Why do we mark a data member transient?
We mark fields as transient to exclude them from serialization when: 1) They contain sensitive data, 2) They're derived/calculated fields, 3) They're runtime-specific (like file handles or threads).
96. Is it allowed to mark a method as transient?
No, transient is only applicable to instance variables. Methods are not part of an object's serialized state, so marking methods transient would be meaningless and causes a compiler error.
97. How does marking a field as transient makes it possible to serialize an object?
By marking non-serializable fields as transient, we allow the rest of the object to be serialized. During deserialization, transient fields get default values (null, 0, false) unless custom readObject() is implemented.
98. What is Externalizable interface in Java?
Externalizable extends Serializable and provides two methods (writeExternal() and readExternal()) for complete control over serialization process, unlike default Java serialization.
99. What is the difference between Serializable and Externalizable interface?
Serializable: Automatic serialization, uses reflection, slower. Externalizable: Manual control, must implement methods, faster, more flexible. Serializable is marker interface, Externalizable has methods.
100. What is Reflection in Java?
Reflection is an API that allows inspection and modification of class structure/behavior at runtime. It can examine fields, methods, constructors, and create objects/invoke methods dynamically.
101. What are the uses of Reflection in Java?
Reflection is used for: 1) IDE features, 2) Debuggers, 3) Testing frameworks, 4) Dependency injection, 5) Class browsers, 6) Serialization, 7) ORM frameworks, 8) Dynamic proxies.
102. How can we access private method of a class from outside the class?
Using Reflection: getDeclaredMethod() then setAccessible(true). This bypasses access control but requires SecurityManager permission. Generally discouraged except for testing frameworks.
103. How can we create an Object dynamically at Runtime in Java?
Using Class.newInstance() or Constructor.newInstance() via Reflection. Also possible through: 1) Cloning, 2) Deserialization, 3) Factory patterns, 4) Dependency injection containers.
104. What is Garbage Collection in Java?
Garbage Collection is JVM's automatic memory management that reclaims memory occupied by unreachable objects. It prevents memory leaks and manual memory management errors.
105. Why Java provides Garbage Collector?
To: 1) Automate memory management, 2) Prevent memory leaks, 3) Avoid dangling pointers, 4) Reduce programmer burden, 5) Improve program safety and reliability.
106. What is the purpose of gc() in Java?
System.gc() suggests JVM to run garbage collection, but doesn't guarantee it. It's a hint that more memory might be available if GC runs. Generally should be avoided in production code.
107. How does Garbage Collection work in Java?
GC works by: 1) Marking reachable objects (from GC roots), 2) Sweeping unreachable objects, 3) Compacting memory (optional). Modern JVMs use generational collection with different algorithms (Mark-Sweep, Mark-Compact, Copying).
108. When does an object become eligible for Garbage Collection in Java?
When: 1) No references point to it, 2) All references are null, 3) Reference is only from other garbage objects, 4) Weak/Soft/Phantom references aren't preventing collection, 5) It's in an isolated island of objects.
109. Why do we use finalize() method in Java?
finalize() is called before GC reclaims an object, allowing cleanup of non-memory resources (like file handles). However, it's unreliable (may never be called) and deprecated since Java 9 (use Cleaner instead).
110. What are the different types of References in Java?
1) Strong (normal references), 2) Soft (cleared when memory is low), 3) Weak (cleared when GC runs), 4) Phantom (enqueued after finalization). Soft/Weak/Phantom are in java.lang.ref package.
111. How can we reference an unreferenced object again?
Once an object is unreachable, it cannot be referenced again normally. However, during finalize(), the object can make itself reachable again by assigning 'this' to a static field (object resurrection), though this is bad practice.
112. What kind of process is the Garbage collector thread?
GC runs in a daemon thread (for most collectors). It's typically low-priority but can become high-priority when memory is low. Modern JVMs have sophisticated multi-threaded collectors (like G1, ZGC).
113. What is the purpose of the Runtime class?
Runtime provides: 1) Interface to JVM environment, 2) Memory management (gc(), totalMemory()), 3) Process execution (exec()), 4) Shutdown hooks, 5) Debugging info. Singleton accessed via Runtime.getRuntime().
114. How can we invoke an external process in Java?
Using Runtime.exec() or ProcessBuilder. ProcessBuilder is preferred (since Java 5) as it provides better control over environment, working directory, and I/O streams.
115. What are the uses of Runtime class?
Runtime is used for: 1) Running external processes, 2) Getting memory stats, 3) Adding shutdown hooks, 4) Forcing GC, 5) Getting available processors, 6) Loading dynamic libraries.
116. What is a Nested class?
A class defined within another class. Types: 1) Static nested class, 2) Inner class (non-static), 3) Local class (in method), 4) Anonymous class. Improves encapsulation and organization.
117. How many types of Nested classes are in Java?
Four types: 1) Static nested classes, 2) Non-static inner classes, 3) Local classes (defined in methods/blocks), 4) Anonymous classes (unnamed one-time use classes).
118. Why do we use Nested Classes?
To: 1) Group related classes, 2) Increase encapsulation, 3) Improve readability, 4) Access outer class members, 5) Create helper classes, 6) Implement callbacks, 7) Reduce namespace pollution.
119. What is the difference between a Nested class and an Inner class in Java?
"Nested class" is umbrella term for all classes defined within another class. "Inner class" specifically means non-static nested class that has access to outer class instance members.
120. What is a Nested interface?
An interface defined within another class or interface. By default static (even without keyword). Used to group related interfaces and improve encapsulation. Example: Map.Entry in Java Collections.
121. How can we access the non-final local variable, inside a Local Inner class?
Since Java 8, local variables accessed in inner classes only need to be effectively final (not necessarily declared final). The value cannot be changed after initialization.
122. Can an Interface be defined in a Class?
Yes, interfaces can be nested inside classes (static by default). Example: class Outer { interface Nested { } }. The interface can be public, private, or package-private.
123. Do we have to explicitly mark a Nested Interface public static?
No, nested interfaces are implicitly static (whether declared inside class or another interface). The static modifier can be included but is redundant. Access modifiers are optional.
124. Why do we use Static Nested interface in Java?
Static nested interfaces are used to: 1) Group related interfaces, 2) Improve encapsulation, 3) Provide namespace separation, 4) Create helper interfaces, 5) Implement design patterns like Builder.
125. What is the meaning of Immutable in the context of String class in Java?
Immutable means String objects cannot be changed after creation. Any modification creates a new String. This ensures thread-safety, security in class loading, and enables string pooling.
126. Why a String object is considered immutable in java?
Strings are immutable because: 1) Security (prevent tampering), 2) Thread-safety, 3) Caching (string pool), 4) Hashcode caching, 5) Class loading safety, 6) Performance optimization.
127. How many objects does following code create? String s = new String("hello");
Creates 2 objects: 1) String literal "hello" in string pool (if not already present), 2) New String object in heap via constructor. The reference 's' points to the heap object.
128. How many ways are there in Java to create a String object?
Main ways: 1) String literals (String s = "hello";), 2) new keyword (String s = new String("hello");), 3) char arrays, 4) StringBuffer/StringBuilder, 5) Concatenation (+ operator), 6) valueOf() methods.
129. How many objects does following code create? String s1 = "hello"; String s2 = "hello";
Creates 1 object. Both references point to the same "hello" literal in string pool. JVM reuses string literals to optimize memory usage.
130. What is String interning?
String interning is JVM's optimization that stores only one copy of each distinct string literal in string pool. intern() method adds runtime strings to pool. Saves memory for duplicate strings.
131. Why Java uses String literal concept?
String literals enable: 1) Memory efficiency (pooling), 2) Performance (quick comparison via ==), 3) Immutability safety, 4) Compile-time optimization, 5) Security in class loading.
132. What is the basic difference between a String and StringBuffer object?
String is immutable (thread-safe but creates new objects on modification). StringBuffer is mutable (modifies existing object), thread-safe (synchronized), slower than StringBuilder.
133. How will you create an immutable class in Java?
To create immutable class: 1) Make class final, 2) Make fields private final, 3) No setter methods, 4) Initialize all fields in constructor, 5) Return deep copies of mutable fields, 6) Don't leak 'this'.
134. What is the use of toString() method in java?
toString() provides string representation of an object. Used by: 1) System.out.println(), 2) String concatenation, 3) Debugging, 4) Logging. Default implementation returns className@hashCode.
135. Arrange the three classes String, StringBuffer and StringBuilder in the order of efficiency for String processing operations?
StringBuilder (fastest, non-sync) > StringBuffer (sync, thread-safe) > String (slowest for modifications due to immutability). For concatenation in loops, StringBuilder is 1000x faster than String.
136. What is Exception Handling in Java?
Exception handling manages runtime errors using: try (risky code), catch (handle exceptions), finally (cleanup), throw (raise exception), throws (declare exceptions). Prevents program termination on errors.
137. In Java, what are the differences between a Checked and Unchecked?
Checked exceptions: Must be declared/caught (IOException, SQLException). Unchecked (RuntimeException): Optional handling (NullPointerException, ArrayIndexOutOfBounds). Errors (unchecked): Serious JVM failures.
138. What is the base class for Error and Exception classes in Java?
Throwable is the base class for both Error and Exception. Error indicates serious JVM failures (OutOfMemoryError). Exception covers recoverable conditions (IOException, RuntimeException).
139. What is a finally block in Java?

StringJoiner: Utility to join strings with delimiter, prefix and suffix. Used internally by String.join() and Collectors.joining(). Example: new StringJoiner(",").add("a").add("b").toString()
410. What is the type of a Lambda expression in Java 8?
Lambda's type is a functional interface - interface with single abstract method (e.g., Runnable, Predicate). The target type context determines the lambda's type. Same lambda can have different types in different contexts
411. What is the target type of a lambda expression?
Target type is the type the lambda is being converted to (functional interface). Determined by: 1) Variable declaration, 2) Method parameter, 3) Return type, 4) Cast expression. Enables type inference
412. What are the main differences between an interface with default method and an abstract class in Java 8?
Interface: 1) Multiple inheritance, 2) No instance fields, 3) Default methods only, 4) All methods public. Abstract class: 1) Single inheritance, 2) Can have state, 3) Can have any visibility, 4) Constructors
413. What is the Stream API in Java?
Stream API: Sequence of elements supporting functional-style operations (filter/map/reduce). Features: 1) No storage, 2) Lazy evaluation, 3) Parallel processing, 4) Internal iteration. Used with collections, arrays, I/O
414. What are the main advantages of using the Stream API?
Advantages: 1) Declarative style, 2) Parallel processing easy, 3) Reduced boilerplate, 4) Lazy evaluation, 5) Method chaining, 6) Immutable data, 7) Better readability, 8) Functional programming support
415. How do you create a stream?
Ways to create streams: 1) Collection.stream(), 2) Arrays.stream(), 3) Stream.of(values), 4) Stream.iterate/generate, 5) Files.lines(), 6) IntStream.range(), 7) Pattern.splitAsStream()
416. What are intermediate and terminal operations?
Intermediate ops (filter, map): 1) Lazy, return Stream, build pipeline. Terminal ops (collect, forEach): 1) Eager, trigger processing, 2) Return non-Stream or void, 3) Close stream
417. What is the map() operation?
map(): Intermediate op that transforms each element using Function. Returns new Stream. Example: names.stream().map(String::toUpperCase). Original stream unchanged. For 1:1 element transformation
418. How does filtering work in streams?
filter(): Intermediate op using Predicate. Keeps elements where predicate is true. Example: numbers.stream().filter(n -> n%2==0). Lazy - only evaluates when terminal op executes
419. What is the difference between collect() and reduce()?
collect(): 1) Mutable reduction, 2) Uses Collector, 3) Better for collections. reduce(): 1) Immutable reduction, 2) Returns Optional, 3) Associative ops only. collect() more flexible for complex cases
420. Can you perform parallel processing with the Stream API?
Yes, via: 1) parallelStream() on collections, 2) parallel() on existing stream. Uses ForkJoinPool.commonPool(). Considerations: 1) Thread-safety, 2) Stateless ops, 3) Avoid blocking ops, 4) Measure performance
421. What is the purpose of the flatMap() operation?
flatMap(): 1) Maps each element to stream, 2) Flattens resulting streams into one stream. For 1:N transformations. Example: words.stream().flatMap(word -> letters(word)) - stream of all letters
422. How can you limit the number of elements in a stream?
Use limit(n): Short-circuiting intermediate op. Returns stream with first n elements. Example: Stream.iterate(1, i -> i+1).limit(10) - numbers 1-10. Efficient with infinite streams
423. What does skip() do in a stream?
skip(n): Intermediate op that discards first n elements. Returns stream of remaining elements. Example: Stream.of(1,2,3,4).skip(2) - [3,4]. Often used with limit() for pagination
424. What are primitive streams?
Specialized streams: IntStream, LongStream, DoubleStream. Avoid boxing overhead. Created via: 1) Arrays.stream(int[]), 2) IntStream.range(), 3) mapToInt() etc. Have specialized ops (sum(), average())
425. What is the difference between findFirst() and findAny()?
findFirst(): Returns first element (stable in parallel). findAny(): Returns any element (more parallel-friendly). Both return Optional. Prefer findAny() for parallel when order doesn't matter
426. What is the purpose of the peek() operation?
peek(): Intermediate op for debugging - performs action without modifying stream. Example: .peek(System.out::println). Should be non-interfering. Not for business logic - use map()/forEach() instead
427. How do you combine multiple streams?
Combine via: 1) Stream.concat(a, b) - two streams, 2) flatMap() - nested streams, 3) Stream.of(s1, s2).flatMap(s -> s) - multiple streams. Note: concat() creates new stream, doesn't modify inputs
428. What is the allMatch(), anyMatch(), and noneMatch()?
Terminal predicate ops: 1) allMatch(Predicate) - all elements match, 2) anyMatch() - at least one matches, 3) noneMatch() - no elements match. Short-circuiting (may not process all elements)
429. What is the Collectors class?
Collectors: Factory for Collector implementations used with Stream.collect(). Common collectors: toList(), toSet(), toMap(), joining(), groupingBy(), partitioningBy(), reducing(), summarizingInt() etc.
430. How do you use toList()?
Collect stream to List: stream.collect(Collectors.toList()). Since Java 16: stream.toList() (returns unmodifiable list). For ArrayList: collect(Collectors.toCollection(ArrayList::new))
431. How can you use toMap() to create a map from a stream?
toMap() variants: 1) toMap(keyMapper, valueMapper), 2) with merge function for duplicates, 3) with map supplier. Example: persons.collect(toMap(Person::id, Function.identity(), (p1,p2) -> p1))
432. What does the joining() collector do?
joining(): Concatenates CharSequence elements. Options: 1) joining() - no delimiter, 2) joining(delimiter), 3) joining(delimiter, prefix, suffix). Example: names.collect(joining(", ", "[", "]"))
433. How do you use groupingBy()?
groupingBy(): Creates Map of groups. Variants: 1) groupingBy(classifier), 2) with downstream collector, 3) with map factory. Example: persons.collect(groupingBy(Person::department, counting()))
434. What is partitioningBy()?
partitioningBy(): Special case of groupingBy() with boolean key (two groups). Example: numbers.collect(partitioningBy(n -> n%2==0)) - splits into even/odd. More efficient than groupingBy() for boolean keys
435. What is reducing()?
reducing(): Collector for custom reductions. Variants: 1) reducing(identity, op), 2) reducing(op), 3) reducing(identity, mapper, op). Similar to reduce() but as Collector for use with groupingBy()
